# Frontend 3-Min Interview Script

## 0:00 - 0:25 项目定位

这是我做的 `CryptoFolio`，定位是“前端资产分析与风控仪表盘”。
目标不是只展示价格，而是把资产、交易、收益、风险和告警整合成一个可决策的前端应用。

## 0:25 - 1:00 首屏与信息架构

首页我做了一个 `Hero` 摘要区，先给面试官 4 个关键信号：
- 组合价值
- 持仓币种
- 最大回撤
- 启用告警数

这样用户第一眼就能判断当前组合状态。
下面再分层展示 KPI、收益曲线、交易流水、风险指标和告警中心，信息从“总览”到“细节”逐级展开。

## 1:00 - 1:45 核心功能与业务逻辑

功能上包含：
- 资产 CRUD
- 交易流水（BUY/SELL）
- 成本法切换（加权平均/FIFO）
- 风险指标（最大回撤、集中度、波动率）
- 告警规则（价格、仓位、回撤）

一个重点是成本口径：
我把成本计算收敛到统一函数，切换成本法后，资产列表、KPI、图表都联动更新，避免各模块口径不一致。

## 1:45 - 2:25 前端工程能力

工程上我重点做了三件事：
1. 状态与持久化：资产、交易、快照、告警规则都存 localStorage，并做了脏数据容错。
2. 性能优化：
- 刷新防重入 + 排队刷新
- 价格请求节流
- 成本计算缓存与交易分组缓存
3. 可验证性：
- 独立写了公式测试脚本，覆盖平均法、FIFO、回撤、集中度。

## 2:25 - 3:00 设计思路与总结

视觉上我参考了成熟前端产品的风格：
- 信息层级清晰
- 数据可扫描
- 状态色有语义
- 动效克制但有反馈

这个项目最终体现的是：
我不仅能写页面，还能把“业务规则 + 数据可视化 + 工程稳定性”整合成可演示、可扩展、可测试的前端产品。
